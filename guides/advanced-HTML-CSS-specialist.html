Domain 1: HTML5 Essentials

The web development trifecta: HTML5 (structure), CSS3 (presentation), JavaScript (behavior). These three together are also called the "HTML5 Family."
HTML5 Structure Elements (Semantic Tags)
Key Semantic Elements
<header>Page or section header; introductory content, navigation links
<nav>Major navigation block; links to other pages or sections
<main>Dominant content of the document; only ONE per page
<article>Self-contained content (blog post, news item, comment)
<section>Thematic grouping of content with a heading
<aside>Tangentially related content; sidebar material
<footer>Footer for its nearest sectioning ancestor
<figure> / <figcaption>Self-contained figure with optional caption
<details> / <summary>Disclosure widget; expandable content
<mark>Highlighted/relevant text
<time>Date/time with optional machine-readable datetime attr
<progress>Completion progress of a task

HTML5 Validation
W3C Markup Validator: validator.w3.org — validates HTML5 documents. Identify and fix: missing doctype, unclosed tags, invalid attribute values, deprecated elements.
Always include <!DOCTYPE html> to trigger standards mode (not quirks mode).
Mobile Websites vs. Mobile Apps
Mobile WebsiteAccessible via browser, URL-based, no download required, easier to update, cross-platform
Mobile AppDownloaded/installed, access to device hardware (GPS, camera), works offline, better performance, platform-specific
Universal Markup Creation
  • Write valid, well-formed HTML that works across all browsers and devices
  • Separate structure (HTML), presentation (CSS), and behavior (JS)
  • Use semantic elements for screen readers and SEO
  • Design progressively: basic functionality works without CSS/JS

CSS Positioning
staticDefault. Normal document flow. top/left/etc have no effect.
relativeOffset from its normal position. Still in document flow.
absoluteRemoved from flow. Positioned relative to nearest non-static ancestor.
fixedRemoved from flow. Positioned relative to viewport. Stays on scroll.

Responsive Web Design (RWD)
Three pillars of RWD (Ethan Marcotte):
  1. Fluid grids — use percentages, not fixed pixels
  2. Flexible images — max-width: 100%
  3. Media queries — apply different CSS at different viewport sizes
/* Viewport meta tag — required for mobile RWD */
<meta name="viewport" content="width=device-width, initial-scale=1.0">

/* Media query examples */
@media (max-width: 768px) {
  body { font-size: 14px; }
}

@media (min-width: 1024px) {
  .container { max-width: 960px; }
}

/* Mobile-first approach (preferred) */
/* Base styles target mobile, then expand upward */

Domain 3: Advanced CSS3 Techniques

CSS3 Transforms
/* 2D Transforms */
.box {
  transform: translate(50px, 20px);    /* move */
  transform: rotate(45deg);            /* rotate */
  transform: scale(1.5);               /* scale (both axes) */
  transform: skew(10deg, 5deg);        /* skew */
  transform: matrix(1,0,0,1,50,20);   /* matrix() combines all */
}

/* 3D Transforms */
.box3d {
  transform: rotateX(30deg) rotateY(45deg);
  transform: perspective(500px) translateZ(100px);
  transform-origin: center center;
}

/* Set perspective on parent for children's 3D */
.parent {
  perspective: 1000px;
  transform-style: preserve-3d;
}
CSS3 Transitions
/* Shorthand: property duration timing-function delay */
.btn {
  background: blue;
  transition: background 0.3s ease-in-out 0s;
}
.btn:hover { background: red; }

/* Multiple transitions */
.card {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
Timing functions: ease (default) | linear | ease-in | ease-out | ease-in-out | cubic-bezier()
CSS3 Animations
/* Define keyframes */
@keyframes slideIn {
  0%   { transform: translateX(-100px); opacity: 0; }
  100% { transform: translateX(0); opacity: 1; }
}

/* Apply animation */
.element {
  animation-name: slideIn;
  animation-duration: 1s;
  animation-timing-function: ease-out;
  animation-delay: 0.5s;
  animation-iteration-count: infinite;  /* or a number */
  animation-direction: alternate;         /* normal | reverse | alternate */
  animation-fill-mode: forwards;           /* none | forwards | backwards | both */
  /* Shorthand: */
  animation: slideIn 1s ease-out 0.5s infinite alternate forwards;
}
SASS (Syntactically Awesome Style Sheets)
SASS is a CSS preprocessor — it extends CSS with features not available natively, then compiles to standard CSS.

Key features: Variables ($primary: #333), Nesting, Mixins, Partials & Imports, Inheritance (@extend), Operators (math in CSS)
SASS is NOT a browser technology — it requires compilation to CSS before use.

Operators & Control Flow
// Comparison operators
==   // equality (loose — type coercion)
===  // strict equality (no coercion) ← PREFER THIS
!=   // not equal (loose)
!==  // strict not equal

// Conditional
if (score >= 75) {
  console.log("Pass");
} else if (score >= 60) {
  console.log("Near miss");
} else {
  console.log("Fail");
}

// Ternary
let result = (score >= 75) ? "Pass" : "Fail";

// Loops
for (let i = 0; i < 10; i++) { ... }
while (condition) { ... }
do { ... } while (condition);
for (let item of array) { ... }   // iterates values
for (let key in object) { ... }   // iterates keys
Arrays
const arr = [1, 2, 3];
arr.push(4);         // add to end
arr.pop();           // remove from end
arr.shift();         // remove from front
arr.unshift(0);     // add to front
arr.length;          // number of elements
arr.map(x => x*2);   // transform each element
arr.filter(x => x>2); // filter elements
arr.forEach(fn);     // iterate (no return)
arr.find(fn);        // first matching element
arr.indexOf(val);    // find index

Domain 5: JS Events, Functions & Methods

Event Handlers
// Method 1: HTML attribute (avoid)
<button onclick="handleClick()">Click</button>

// Method 2: DOM property
document.getElementById("btn").onclick = handleClick;

// Method 3: addEventListener (preferred)
element.addEventListener("click", handleClick);
element.addEventListener("click", handleClick, false); // bubble phase

// Common events
click | dblclick | mouseover | mouseout | mouseenter
keydown | keyup | keypress
load | DOMContentLoaded | resize | scroll
focus | blur | change | submit | input
Functions
// Function declaration (hoisted)
function greet(name) {
  return `Hello, ${name}!`;
}

// Function expression (NOT hoisted)
const greet = function(name) { return `Hello, ${name}!`; };

// Arrow function (ES6) — no own 'this'
const greet = (name) => `Hello, ${name}!`;

// Closures — inner function retains access to outer scope
function counter() {
  let count = 0;
  return function() { return ++count; };
}
Promises & Async/Await
// Promise
const p = new Promise((resolve, reject) => {
  // async work...
  resolve("success"); // or reject(new Error("fail"))
});
p.then(val => console.log(val))
 .catch(err => console.error(err));

// Async/Await (syntactic sugar over Promises)
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
  } catch (err) {
    console.error(err);
  }
}
Modules (ES6): Use export / import to share code across files. Script tags need type="module".

JavaScript Form Validation
const form = document.getElementById("myForm");
form.addEventListener("submit", function(event) {
  const email = document.getElementById("email").value;
  const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
  
  if (!regex.test(email)) {
    event.preventDefault();  // stop submission
    alert("Invalid email address");
  }
});

// Constraint Validation API
input.validity.valid        // true if all constraints pass
input.validity.valueMissing  // true if required and empty
input.validity.patternMismatch

Always validate on the server side too — client-side validation can be bypassed!

Domain 9: Copyright, Ethics & Legal

Accessibility Standards
ADA (Americans with Disabilities Act)Requires websites of public accommodations to be accessible. No specific technical standard mandated federally, but WCAG 2.1 AA is widely referenced.
Section 508Federal law requiring US government agencies to make electronic and IT accessible to people with disabilities. References WCAG 2.0 AA.
WCAGWeb Content Accessibility Guidelines by W3C. Levels: A, AA, AAA. Four principles: Perceivable, Operable, Understandable, Robust (POUR)
ARIAAccessible Rich Internet Applications — HTML attributes (role, aria-label, aria-hidden) that improve screen reader support
Privacy Laws
Law/RegulationScopeKey Points
GDPREU residentsConsent required; right to erasure; data portability; notify breaches within 72 hrs
HIPAAUS healthcareProtects PHI (Protected Health Information); applies to covered entities
COPPAUS children onlineParental consent required for data collection from children under 13
CCPACalifornia residentsRight to know, delete, and opt-out of data sale
Copyright in Web Development
  • Original creative works are automatically copyrighted — no registration required
  • Fair use allows limited use for commentary, education, news, parody
  • Creative Commons licenses allow controlled sharing
  • Always obtain licenses for stock photos, fonts, and icons used commercially
  • Open-source licenses (MIT, GPL, Apache) have specific requirements
Web Security Basics
  • HTTPS — encrypts data in transit via TLS/SSL; required for forms handling sensitive data
  • XSS (Cross-Site Scripting) — attacker injects malicious scripts; sanitize user input
  • CSRF (Cross-Site Request Forgery) — use tokens; check referrer headers
  • SQL Injection — use parameterized queries, never concatenate user input
  • Content Security Policy (CSP) — HTTP header restricting resource origins
